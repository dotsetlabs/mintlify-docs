# deadfall metrics

View statistics about trap triggers and honeypot activity.

## Usage

```bash
deadfall metrics [options]
```

## Options

| Option | Description |
|--------|-------------|
| `--reset` | Reset all metrics to zero |
| `--json` | Output as JSON |

## Examples

```bash
# View metrics
deadfall metrics

# View as JSON
deadfall metrics --json

# Reset all metrics
deadfall metrics --reset
```

## Reset

The `--reset` flag clears all collected metrics:

```bash
$ deadfall metrics --reset

Metrics reset successfully.
```

This is useful for:
- Starting fresh after testing
- Beginning a new monitoring period
- Clearing test data before production use

## Use Cases

### Security Monitoring

Track how often AI agents trigger traps to detect:
- Unauthorized file access
- Compromised agents
- Suspicious behavior patterns

### Testing Traps

After creating new traps, use metrics to verify they're working:

```bash
# Create trap
deadfall trap context CONTEXT.md

# Test with AI agent
# ...

# Check metrics
deadfall metrics
```

### Compliance

Export metrics as JSON for audit trails:

```bash
deadfall metrics --json > metrics-$(date +%Y%m%d).json
```
