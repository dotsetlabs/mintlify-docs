# CLI Reference

Complete reference for all dotset commands.

## Installation

```bash
npm install -g @dotsetlabs/cli
```

---

## Core Commands

### `dotset init`

Initialize a new dotset project.

```bash
dotset init [options]
```

| Option | Description |
|:-------|:------------|
| `--name <name>` | Project name |
| `--cloud` | Create cloud project immediately |
| `--axion` | Enable Secrets module |
| `--gluon` | Enable Security module |
| `--hadron` | Enable Local CI module |
| `--tachyon` | Enable Tunnels module |

### `dotset run`

Run a command with secrets injected and security monitoring enabled.

```bash
dotset run [options] -- <command>
```

| Option | Description |
|:-------|:------------|
| `--scope <env>` | Environment scope: development, staging, production |
| `-s, --service <name>` | Service to scope secrets to |
| `--no-secrets` | Disable secret injection |
| `--no-monitor` | Disable security monitoring |
| `--mode <mode>` | Protection mode: detect, redact, block |

### `dotset login`

Authenticate with the dotset cloud.

```bash
dotset login
```

### `dotset logout`

Clear stored credentials.

```bash
dotset logout
```

### `dotset status`

Show project and authentication status.

```bash
dotset status
```

## Secrets Commands

### `dotset secrets set`

Set a secret value.

```bash
dotset secrets set <key> [value] [options]
```

| Option | Description |
|:-------|:------------|
| `--scope <env>` | Environment scope (default: development) |
| `-s, --service <name>` | Service to scope the secret to |

### `dotset secrets get`

Get a secret value.

```bash
dotset secrets get <key> [options]
```

### `dotset secrets list`

List all secrets.

```bash
dotset secrets list [options]
```

| Option | Description |
|:-------|:------------|
| `--scope <env>` | Environment scope |
| `--show-values` | Show secret values (use with caution) |

### `dotset secrets delete`

Delete a secret.

```bash
dotset secrets delete <key> [options]
```

## Security Commands

### `dotset scan`

Run static security analysis.

```bash
dotset scan [options]
```

| Option | Description |
|:-------|:------------|
| `--fix` | Attempt to fix issues automatically |
| `--json` | Output results as JSON |

### `dotset sbom`

Generate a Software Bill of Materials.

```bash
dotset sbom [options]
```

| Option | Description |
|:-------|:------------|
| `--format <format>` | Output format: cyclonedx, spdx |
| `-o, --output <file>` | Output file path |
| `--static` | Generate from package.json |

## CI Commands

### `dotset ci`

Run GitHub Actions jobs locally.

```bash
dotset ci [jobs...] [options]
```

| Option | Description |
|:-------|:------------|
| `--workflow <path>` | Path to workflow file |
| `--scope <env>` | Environment scope for secrets |
| `--mode <mode>` | Protection mode: detect, redact, block |
| `--list` | List available workflows and jobs |
| `--dry-run` | Preview without executing |
| `--sync` | Sync results to dotset cloud |
| `--no-secrets` | Disable secret injection |
| `--no-monitor` | Disable leak monitoring |

## Tunnel Commands

### `dotset share`

Share localhost via secure tunnel.

```bash
dotset share <port> [options]
```

| Option | Description |
|:-------|:------------|
| `--subdomain <name>` | Request specific subdomain |
| `--inspect` | Enable request inspector |
| `--auth` | Require authentication |
| `--project <id>` | Link to a specific project |

### `dotset tunnels`

List active tunnels.

```bash
dotset tunnels
```

## Team Commands

### `dotset team list`

List all team members.

```bash
dotset team list
```

### `dotset team add`

Add a team member.

```bash
dotset team add <email> [options]
```

| Option | Description |
|:-------|:------------|
| `--role <role>` | Role: admin, member, readonly |
| `--scopes <scopes>` | Comma-separated: development, staging, production |

### `dotset team update`

Update a team member's role or scopes.

```bash
dotset team update <email> [options]
```

| Option | Description |
|:-------|:------------|
| `--role <role>` | New role |
| `--scopes <scopes>` | New scopes |

### `dotset team remove`

Remove a team member.

```bash
dotset team remove <email>
```

## Token Commands

### `dotset tokens list`

List service tokens.

```bash
dotset tokens list
```

### `dotset tokens create`

Create a service token.

```bash
dotset tokens create <name> [options]
```

| Option | Description |
|:-------|:------------|
| `--scopes <scopes>` | Required. Comma-separated scopes |
| `--readonly` | Token can only read secrets |
| `--expires <duration>` | Expiration: 30d, 90d, 1y |

### `dotset tokens revoke`

Revoke a service token.

```bash
dotset tokens revoke <name>
```

## Audit Commands

### `dotset audit`

View audit logs.

```bash
dotset audit [options]
```

| Option | Description |
|:-------|:------------|
| `--action <action>` | Filter by action type |
| `--user <email>` | Filter by user |
| `--scope <scope>` | Filter by environment scope |
| `--source <product>` | Filter by product: axion, gluon, hadron, tachyon |
| `--since <date>` | Start date (YYYY-MM-DD) |
| `--until <date>` | End date (YYYY-MM-DD) |
| `--limit <n>` | Maximum entries to show |

### `dotset audit export`

Export audit logs.

```bash
dotset audit export [options]
```

| Option | Description |
|:-------|:------------|
| `--format <format>` | Output format: json, csv |
| `--output <file>` | Output file path |
| `--since <date>` | Start date |
| `--until <date>` | End date |

### `dotset audit verify`

Verify audit log integrity.

```bash
dotset audit verify
```
