# hardpoint scan

Run security scans on your development environment.

## Usage

```bash
hardpoint scan [scanner...] [flags]
```

## Available Scanners

| Scanner | Description |
| ------- | ----------- |
| `ai` | Scan AI configuration files (mcp.json, .cursorrules, CLAUDE.md) |
| `shell` | Scan shell configurations for backdoors |
| `git` | Check git hooks integrity |
| `network` | Detect exposed localhost services |
| `secrets` | Find secrets in configuration files |

## Flags

### General Flags

| Flag | Short | Description |
| ---- | ----- | ----------- |
| `--path` | `-p` | Path to scan (default: home directory) |
| `--severity` | `-s` | Minimum severity: `info`, `low`, `medium`, `high`, `critical` |
| `--confidence` | `-c` | Minimum confidence: `low`, `medium`, `high` |
| `--exclude` | `-e` | Paths to exclude (can be repeated) |
| `--disable-rules` | | Rule IDs to disable (can be repeated) |
| `--enable-rules` | | Only enable these rules, disable all others (can be repeated) |

### Codebase Scanning Flags (AI Scanner)

| Flag | Description |
| ---- | ----------- |
| `--scope` | Scan scope: `config` (AI files only) or `codebase` (all source files) |
| `--max-depth` | Maximum directory depth for codebase scan (0=unlimited) |
| `--include` | File patterns to include (e.g., `*.py,*.js`) |
| `--ai-exclude` | Additional patterns to exclude from codebase scan |
| `--workers` | Number of parallel scan workers (0=auto) |
| `--max-file-size` | Maximum file size to scan (e.g., `10MB`) |

### Dependency Scanning Flags (AI Scanner)

| Flag | Description |
| ---- | ----------- |
| `--include-deps` | Include dependency directories (node_modules, vendor, etc.) |
| `--deps-only` | Only scan dependency directories |

## Examples

### Run All Scanners

```bash
hardpoint scan
```

### Run Specific Scanners

```bash
# Only AI scanner
hardpoint scan ai

# Multiple scanners
hardpoint scan ai shell secrets
```

### Scan a Project Directory

```bash
hardpoint scan --path /path/to/project
```

### Filter by Severity

```bash
# Only high and critical findings
hardpoint scan --severity high
```

### Exclude Paths

```bash
hardpoint scan --exclude node_modules --exclude .git
```

### Disable Specific Rules

```bash
# Skip specific rules by ID
hardpoint scan --disable-rules AI-001 --disable-rules SHELL-002

# Multiple rules in one flag
hardpoint scan --disable-rules AI-001,SECRET-003
```

### Enable Only Specific Rules

```bash
# Only run AI rules
hardpoint scan --enable-rules AI-001 --enable-rules AI-002

# Useful for focused scanning
hardpoint scan ai --enable-rules AI-001
```

### Output Formats

```bash
# JSON output
hardpoint scan --output json

# SARIF for GitHub Code Scanning
hardpoint scan --output sarif > results.sarif
```

### Codebase Scanning

Scan your entire codebase for prompt injection hidden in comments and documentation:

```bash
# Scan all source code comments
hardpoint scan ai --scope codebase

# Limit scanning depth
hardpoint scan ai --scope codebase --max-depth 5

# Only scan specific file types
hardpoint scan ai --scope codebase --include "*.py,*.js,*.ts"

# Scan with more workers for large codebases
hardpoint scan ai --scope codebase --workers 8
```

### Dependency Scanning

Scan dependencies for supply chain attacks:

```bash
# Include dependencies in scan
hardpoint scan ai --scope codebase --include-deps

# Scan only dependencies (node_modules, vendor, etc.)
hardpoint scan ai --deps-only

# Combine with codebase scanning
hardpoint scan ai --scope codebase --include-deps --workers 4
```
