# dotset labs

**One CLI for secrets, security, CI, and crash replay.**

dotset is a unified developer platform that combines encrypted secrets management, runtime security, local CI debugging, and instant crash reproduction into a single developer experience.

## Quick Install

```bash
npm install -g @dotsetlabs/cli
```

## The Unified Runtime

With one CLI, manage your entire development security stack:

```bash
# Initialize your project
dotset init

# Run with secrets + protection
dotset run --scope prod --mode block -- npm start

# Debug CI locally
dotset ci build test --scope prod
```

## Core Commands

| Command | Description |
|:--------|:------------|
| `dotset init` | Initialize a new project |
| `dotset run` | Run with secrets + monitoring |
| `dotset ci` | Run CI jobs locally |
| `dotset replay` | Replay crashes locally |
| `dotset login` | Authenticate with cloud |
| `dotset status` | Show project status |

## Module Commands

```bash
# Secrets
dotset secrets set API_KEY "sk-..."
dotset secrets list

# Security
dotset scan
dotset sbom --static

# Local CI
dotset ci build test
dotset ci --list

# Crash Replay (Lagrangian)
dotset replay <event-id>

# Team & Governance
dotset team add user@example.com --role member
dotset audit
dotset audit verify
```

## Why dotset?

Most security tools are:
- **Siloed** — Separate tools for secrets, scanning, CI, and tunneling
- **Friction-heavy** — Require extensive configuration
- **Dev-unfriendly** — Designed for ops, not developers

dotset is different:
- **Unified** — One CLI, one config, one workflow
- **Zero-config** — Sensible defaults that just work
- **Dev-first** — Built for the local development workflow
