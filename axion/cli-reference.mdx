# Axion CLI Reference

> **Recommended**: Use the unified `dotset` CLI for the full platform experience. The standalone `axn` command is available when installing `@dotsetlabs/axion` directly.

## Secrets Commands

### dotset secrets init

Initialize the Axion secrets vault.

```bash
dotset secrets init
```

### dotset secrets set

Set an environment variable.

```bash
dotset secrets set <key> [value] [options]
```

| Option | Description |
|--------|-------------|
| `--scope <env>` | Environment scope: `development`, `staging`, `production` |
| `-s, --service <name>` | Scope to a specific service |

**Examples:**

```bash
dotset secrets set DATABASE_URL "postgres://localhost/db"
dotset secrets set API_KEY "sk-xxx" --scope production
dotset secrets set DATABASE_URL "postgres://{{PG_USER}}:{{PG_PASS}}@localhost/db"
```

<Note>
You can pipe values: `echo "secret-value" | dotset secrets set API_KEY`
</Note>

### dotset secrets get

Get an environment variable value.

```bash
dotset secrets get <key> [options]
```

| Option | Description |
|--------|-------------|
| `--scope <env>` | Environment scope (default: `development`) |

### dotset secrets delete

Remove an environment variable.

```bash
dotset secrets delete <key> [options]
```

### dotset secrets list

List all environment variables.

```bash
dotset secrets list [options]
```

| Option | Description |
|--------|-------------|
| `--scope <env>` | Environment scope |
| `--show-values` | Show actual values |

### dotset secrets export

Export secrets to file or stdout.

```bash
dotset secrets export [options]
```

| Option | Description |
|--------|-------------|
| `--scope <env>` | Environment scope |
| `-o, --output <file>` | Write to file |

### dotset secrets import

Import secrets from a .env file.

```bash
dotset secrets import <file> [options]
```

| Option | Description |
|--------|-------------|
| `--scope <env>` | Environment scope |

## Key Management

### dotset secrets key show

Display the encryption key.

```bash
dotset secrets key show
```

### dotset secrets key fingerprint

Show key fingerprint (safe to share).

```bash
dotset secrets key fingerprint
```

### dotset secrets rotate

Rotate encryption key and re-encrypt all secrets.

```bash
dotset secrets rotate
```

## Cloud Sync

### dotset sync push

Push local secrets to cloud.

```bash
dotset sync push [options]
```

| Option | Description |
|--------|-------------|
| `--from-env <file>` | Import from .env file before pushing |

### dotset sync pull

Pull secrets from cloud.

```bash
dotset sync pull
```

### dotset sync status

Check sync status and detect drift.

```bash
dotset sync status
```

### dotset drift

Detect differences between local and cloud.

```bash
dotset drift [options]
```

| Option | Description |
|--------|-------------|
| `--json` | Output as JSON |
| `--ci` | Exit with code 1 if drift detected |

## Running with Secrets

### dotset run

Run a command with secrets injected.

```bash
dotset run [options] -- <command>
```

| Option | Description |
|--------|-------------|
| `--scope <env>` | Environment scope (default: `development`) |
| `-s, --service <name>` | Service to load secrets for |
| `--mode <mode>` | Protection mode: `detect`, `redact`, `block` |

**Examples:**

```bash
dotset run -- npm start
dotset run --scope production -- node dist/index.js
dotset run -s api -- npm start
```
