# Cloud Sync

Axion Cloud enables secure team collaboration with encrypted secret sharing.

## Enable Cloud Sync

```bash
# 1. Sign in
dotset login

# 2. Initialize cloud project
dotset init --cloud

# 3. Push local secrets
dotset sync push
```

## Pushing and Pulling

### Push to Cloud

```bash
# Push all local secrets
dotset sync push

# Import from .env and push in one step
dotset sync push --from-env .env.production
```

### Pull from Cloud

```bash
dotset sync pull
```

### Check Status

```bash
dotset sync status
```

## Drift Detection

Identify differences between local and cloud:

```bash
dotset drift
```

Output shows:
- **Local only** — Secrets not in cloud
- **Cloud only** — Secrets not locally
- **Modified** — Different values

### CI Mode

```bash
dotset drift --ci
```

Exits with code 1 if drift is detected, useful for CI pipelines.

### JSON Output

```bash
dotset drift --json
```

## Importing .env Files

Import existing .env files into your encrypted manifest:

```bash
# Import to development scope
dotset secrets import .env

# Import to production scope
dotset secrets import .env.production --scope production
```

## CI/CD Deployment

### Service Tokens

Create a read-only token for CI/CD:

```bash
# Create via dashboard at app.dotsetlabs.com
```

### Environment Variable

Set `AXION_TOKEN` in your CI environment:

```bash
AXION_TOKEN=axn_xxxxxxxxxxxxx
```

### Start Command

```bash
npx @dotsetlabs/cli run --scope production -- node dist/index.js
```

<Note>
With a service token, secrets are fetched directly from the cloud. No local manifest needed.
</Note>
