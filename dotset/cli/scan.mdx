# dotset scan

Run a Hardpoint security scan on your developer environment. This is a convenience wrapper around the `hardpoint scan` command.

## Usage

```bash
dotset scan [scanners...] [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `scanners` | Specific scanners to run (optional) |

Available scanners:
- `ai` — AI config files (.cursorrules, mcp.json, CLAUDE.md)
- `shell` — Shell configs (.bashrc, .zshrc)
- `git` — Git hooks and config
- `network` — Exposed localhost services
- `secrets` — Hardcoded credentials

## Options

| Option | Description |
|--------|-------------|
| `-s, --severity <level>` | Minimum severity: `info`, `low`, `medium`, `high`, `critical` |
| `-c, --confidence <level>` | Minimum confidence: `low`, `medium`, `high` |
| `-o, --output <format>` | Output format: `terminal`, `json`, `sarif` |
| `--path <path>` | Path to scan (default: current directory) |
| `--scope <scope>` | Scan scope: `config` (default) or `codebase` |
| `--max-depth <n>` | Maximum directory depth |
| `--workers <n>` | Number of parallel workers |
| `-e, --exclude <pattern>` | Exclude patterns (can be repeated) |
| `--include-deps` | Include dependency directories (node_modules, vendor) |
| `--deps-only` | Only scan dependency directories |
| `--install` | Auto-install Hardpoint if missing |

## Examples

```bash
# Run all scanners
dotset scan

# Run specific scanners
dotset scan ai secrets

# Show only high severity findings
dotset scan -s high

# Show only medium+ confidence findings
dotset scan -c medium

# Output as JSON
dotset scan -o json

# Scan a specific path
dotset scan --path ./src

# Scan entire codebase (not just config files)
dotset scan --scope codebase

# Limit directory depth
dotset scan --max-depth 5

# Use parallel workers
dotset scan --workers 4

# Exclude patterns
dotset scan -e node_modules -e vendor -e "*.test.ts"

# Include dependencies
dotset scan --include-deps

# Scan only dependencies
dotset scan --deps-only

# Auto-install if missing
dotset scan --install
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | No critical findings |
| `1` | Findings detected (check output for details) |
| Other | Scan error |

## Auto-Install

If Hardpoint is not installed, you can use the `--install` flag:

```bash
$ dotset scan --install
Installing Hardpoint...
  Downloading hardpoint v0.1.0 for linux/amd64...
  Installed to /home/user/.dotset/bin/hardpoint

Hardpoint Security Scan
=======================
...
```

Alternatively, run `dotset doctor --install` to install all missing tools at once.
