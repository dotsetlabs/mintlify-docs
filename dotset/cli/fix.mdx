# dotset fix

Automatically fix certain security findings detected by Hardpoint. This command proxies to `hardpoint fix`.

## Usage

```bash
dotset fix [finding-id] [files...] [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `[finding-id]` | Finding ID to fix (e.g., `AI-003`, `SHELL-004`) |
| `[files...]` | Specific files to fix (optional) |

## Options

| Option | Description |
|--------|-------------|
| `--list` | List fixable finding types |
| `--install` | Auto-install Hardpoint if missing |

## Fixable Findings

Not all findings can be auto-fixed. Use `--list` to see available fixes:

| Finding | Description | Fix Action |
|---------|-------------|------------|
| `AI-003` | Overly permissive AI instructions | Add safety constraints |
| `SHELL-004` | Dangerous shell alias | Remove or comment out |
| `SHELL-010` | Insecure PATH configuration | Reorder PATH entries |

## Examples

```bash
# List all fixable findings
dotset fix --list

# Fix all instances of AI-003
dotset fix AI-003

# Fix AI-003 in specific files
dotset fix AI-003 .cursorrules CLAUDE.md

# Fix SHELL-004 in shell configs
dotset fix SHELL-004 ~/.bashrc ~/.zshrc
```

## Output

### List Output

```
Fixable Findings:

  AI-003    Overly permissive AI instructions
            Fix: Add safety constraints to AI config files
            Affected files: 2

  SHELL-004 Dangerous shell alias
            Fix: Comment out dangerous aliases
            Affected files: 1

  SHELL-010 Insecure PATH configuration
            Fix: Move user directories after system directories
            Affected files: 1
```

### Fix Output

```
Fixing AI-003...

  ✓ .cursorrules
    - Added file operation restrictions
    - Added network access limitations

  ✓ CLAUDE.md
    - Added safety constraints section

2 files fixed

Backup created: .hardpoint/backups/2024-01-15T12:00:00/
```

## Safety

- Backups are created before any changes
- Use `dotset scan` after fixing to verify
- Review changes with `git diff`

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Success |
| `1` | Error or no fixable findings |
