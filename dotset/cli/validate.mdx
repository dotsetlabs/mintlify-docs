# dotset validate

Validate a Tollgate configuration file for syntax errors and policy issues. This command proxies to `tollgate validate`.

## Usage

```bash
dotset validate [path] [options]
```

## Arguments

| Argument | Description |
|----------|-------------|
| `[path]` | Config file to validate (default: `tollgate.yaml`) |

## Options

| Option | Description |
|--------|-------------|
| `-s, --server <name>` | Validate specific server only |
| `--json` | Output as JSON |
| `-q, --quiet` | Quiet mode (only output errors) |

## Examples

```bash
# Validate default config
dotset validate

# Validate specific file
dotset validate custom-config.yaml

# Validate only postgres server config
dotset validate -s postgres

# JSON output for automation
dotset validate --json

# Quiet mode for CI/CD
dotset validate -q
```

## Output

### Success

```
✓ Configuration valid: tollgate.yaml

  Servers: 3 configured
    - postgres (5 tool policies)
    - filesystem (3 tool policies)
    - github (8 tool policies)

  Analyzers: sql, filesystem, shell
  Guardrails: promptInjection enabled
```

### Errors

```
✗ Configuration invalid: tollgate.yaml

  Errors:
    - Line 15: Unknown action 'block' (did you mean 'deny'?)
    - Line 23: Server 'postgres' missing required 'command' field
    - Line 31: Invalid risk level 'extreme' (valid: safe, read, write, destructive, dangerous)

  Warnings:
    - Server 'github' has no tool policies defined
```

## Exit Codes

| Code | Meaning |
|------|---------|
| `0` | Configuration valid |
| `1` | Configuration invalid or error |
