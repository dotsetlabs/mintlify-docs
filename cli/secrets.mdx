# dotset secrets

Manage your encrypted secrets.

## Commands

| Command | Description |
|---------|-------------|
| `secrets set <key> <value>` | Set a secret |
| `secrets get <key>` | Get a secret value |
| `secrets list` | List all secret keys |
| `secrets delete <key>` | Delete a secret |
| `secrets sync` | Sync with cloud |

---

## secrets set

Add or update a secret.

```bash
dotset secrets set <key> <value> [options]
```

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `-s, --scope <scope>` | Environment scope | `development` |

### Example

```bash
dotset secrets set DATABASE_URL "postgres://user:pass@localhost/db"
dotset secrets set API_KEY "sk-..." --scope production
```

---

## secrets get

Retrieve a secret value.

```bash
dotset secrets get <key> [options]
```

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `-s, --scope <scope>` | Environment scope | `development` |

### Example


---

## secrets list

List all secret keys (values are not shown).

```bash
dotset secrets list [options]
```

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `-s, --scope <scope>` | Environment scope | `development` |

### Example

```bash
$ dotset secrets list

Secrets [development]:
  API_KEY
  DATABASE_URL
```

---

```bash

```

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `-s, --scope <scope>` | Environment scope | `development` |

### Example

```bash

```

---

## secrets sync

Sync secrets with the cloud (requires authentication).

```bash
dotset secrets sync [options]
```

### Options

| Option | Description |
|--------|-------------|
| `--push` | Push local secrets to cloud |
| `--pull` | Pull cloud secrets to local |

### Example

```bash
# Push local to cloud
dotset secrets sync --push

# Pull cloud to local
dotset secrets sync --pull
```

<Note>
Cloud sync requires authentication. Run `dotset login` first.
</Note>
