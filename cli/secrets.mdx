# dotset secrets

Manage your encrypted secrets stored locally in Dotset's native encrypted storage.

## Commands

| Command | Description |
|---------|-------------|
| `secrets init` | Initialize encrypted storage |
| `secrets set <key> <value>` | Set a secret |
| `secrets get <key>` | Get a secret value |
| `secrets list` | List all secret keys |
| `secrets delete <key>` | Delete a secret |

---

## secrets init

Initialize a new encrypted secrets store in the current directory.

```bash
dotset secrets init
```

This creates:
- `.dotset/key` — Your encryption key (**Never commit!**)
- `.dotset/secrets.enc` — Your encrypted secrets

### Output

```bash
$ dotset secrets init

Initializing dotset encrypted storage...

✓ Dotset secrets initialized

Your encryption key:
  dss_xxxxxxxxxxxxxxxxxxxxxxxxxxxx

⚠ Store this key securely — you cannot recover secrets without it.
```

## secrets set

Add or update a secret in the local encrypted store.

```bash
dotset secrets set <key> <value> [options]
```

### Options

| Option | Description | Default |
|--------|-------------|---------|
| `-s, --scope <scope>` | Environment scope | `development` |

### Example

```bash
dotset secrets set API_KEY "sk-12345..."
dotset secrets set DATABASE_URL "postgres://..." --scope production
```

## Storage Location

Secrets are stored locally in your project:

| File | Purpose |
|------|---------|
| `.dotset/key` | Encryption key (add to `.gitignore`) |
| `.dotset/secrets.enc` | Encrypted secrets file |

<Warning>
Add `.dotset/` to your `.gitignore` to prevent accidentally committing your encryption key.
</Warning>

## Using Secrets

Run commands with automatic secret injection:

```bash
dotset run -- npm start
```

This loads all secrets from the native store and injects them as environment variables.
