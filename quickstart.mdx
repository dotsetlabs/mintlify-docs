# Quickstart

This guide gets you from zero to protected in under a minute.

---

## Hardpoint

Hardpoint scans your developer environment for AI-era security threats.

### 1. Install

```bash
go install github.com/dotsetlabs/hardpoint/cmd/hardpoint@latest
```

### 2. Run a Scan

```bash
hardpoint scan
```

### 3. Example Output

```bash
$ hardpoint scan

Hardpoint Security Scan
=======================

Scanning: /Users/dev
Scanners: ai, shell, git, network, secrets

Findings
--------

  CRITICAL  AI-001: Instruction Override Detected
            File: .cursorrules:15
            Pattern attempts to override AI safety guidelines

  HIGH      AI-002: Hidden Unicode Characters
            File: CLAUDE.md:8
            Zero-width characters detected

Summary: 1 critical, 1 high (2 total)
```

### 4. Scanners

| Scanner | Description |
|:--------|:------------|
| `ai` | AI config files (.cursorrules, mcp.json, CLAUDE.md) |
| `shell` | Shell configs (.bashrc, .zshrc) |
| `git` | Git hooks and config |
| `network` | Exposed localhost services |
| `secrets` | Hardcoded credentials |

```bash
# Run specific scanners
hardpoint scan ai secrets
```

### 5. Auto-Fix

Some findings can be fixed automatically:

```bash
# Strip hidden Unicode characters
hardpoint fix AI-003 CLAUDE.md
```

## Tollgate

Tollgate adds policy-based access control to MCP servers used by AI agents.

### 1. Install

```bash
npm install -g @dotsetlabs/tollgate
```

### 2. Wrap Any MCP Server

Instantly protect any MCP server without configuration:

```bash
tollgate wrap npx @modelcontextprotocol/server-postgres
```

This prompts you for approval on every tool call.

### 3. Default Actions

| Action | Behavior | Use Case |
|:-------|:---------|:---------|
| `allow` | Proceed without approval | Trusted operations |
| `deny` | Block entirely | Dangerous operations |
| `prompt` | Ask user (default) | **Recommended** |

```bash
# Allow reads, prompt for everything else
tollgate wrap -d prompt npx @anthropic/mcp-server-filesystem ./src
```

### 4. Use with Claude Desktop

Update your Claude Desktop config:

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": [
        "-y", "@dotsetlabs/tollgate", "wrap",
        "npx", "-y", "@modelcontextprotocol/server-postgres"
      ],
      "env": {
        "DATABASE_URL": "postgres://localhost/mydb"
      }
    }
  }
}
```

### 5. Create a Policy File (Optional)

For fine-grained control:

```bash
# Generate config
tollgate init

# Start with config
tollgate start -s postgres
```

## Deadfall

Deadfall detects AI agent compromise by exploiting their instruction-following behavior. Trapped files instruct agents to call a verification tool — triggering an alert.

### 1. Install

```bash
go install github.com/dotsetlabs/deadfall/cmd/deadfall@latest
```

### 2. Initialize

```bash
deadfall init
```

This creates `deadfall.json` and updates `.gitignore`.

### 3. Create Trap Files

```bash
# AI-specific traps (generate complete files)
deadfall trap cursor-rules          # .cursorrules for Cursor
deadfall trap claude-context        # CLAUDE.md for Claude
deadfall trap copilot-instructions  # .github/copilot-instructions.md
deadfall trap mcp-config            # mcp.json honeypot
```

### 4. Configure MCP Client

Add Deadfall's Honey-MCP server to your MCP client:

```bash
# Auto-configure Claude Desktop and Cursor
deadfall init --mcp
```

### 5. Start the Server

```bash
deadfall serve
```

### 6. Example Alert

When an AI reads a trapped file:

```bash
[ALERT] Trap Triggered
  File: .cursorrules
  Type: cursor-rules
  Token: abc123def456...
  Time: 2026-01-05T14:32:15

✓ Desktop notification sent
✓ Logged to deadfall_alerts.log
```

### 7. Trap Types

| Type | Default Path | Target |
|:-----|:-------------|:-------|
| `cursor-rules` | `.cursorrules` | Cursor AI |
| `claude-context` | `CLAUDE.md` | Claude Code |
| `copilot-instructions` | `.github/copilot-instructions.md` | GitHub Copilot |
| `mcp-config` | `mcp.json` | MCP clients |
| `context` | `CONTEXT.md` | Generic context |
| `code` | `main.go` | Source code |
| `env` | `.env.example` | Environment files |

## Defense in Depth

Use all three tools together for complete protection:

```
   SCAN                  CONTROL                 DETECT
     │                      │                      │
     ▼                      ▼                      ▼
┌─────────┐           ┌─────────┐           ┌──────────┐
│Hardpoint│     →     │Tollgate │     →     │ Deadfall │
│  scan   │           │  wrap   │           │  serve   │
└─────────┘           └─────────┘           └──────────┘
     │                      │                      │
     ▼                      ▼                      ▼
  Detect                Enforce                 Detect
  threats              policies               compromise
```

```bash
# Complete security workflow
hardpoint scan                                    # Scan environment
tollgate wrap npx server                          # Protect with Tollgate
deadfall trap cursor-rules && deadfall serve     # Detect AI compromise
```
