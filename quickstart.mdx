# Quickstart

This guide gets you from zero to protected in under a minute.

## 1. Install

```bash
npm install -g @dotsetlabs/shield
```

## 2. Initialize

Create a new shield project in your repository:

```bash
dotset init
```

This creates:
- `.dotset/key` ‚Äî Your encryption key (add to `.gitignore`)
- `.dotset/manifest.json` ‚Äî Encrypted secrets store (safe to commit)

<Warning>
Never commit your `.dotset/key` file. Add it to `.gitignore` immediately.
</Warning>

## 3. Add Secrets

```bash
dotset secrets set DATABASE_URL "postgres://user:pass@localhost/db"
dotset secrets set API_KEY "sk-..."
```

Secrets are encrypted locally with AES-256-GCM. Even if someone reads your manifest, they can't decrypt without your key.

## 4. Run with Protection

```bash
# Default: detect mode (logs exposures)
dotset run -- npm start

# Recommended: redact mode (replaces secrets)
dotset run --mode redact -- npm start

# Strict: block mode (suppresses output)
dotset run --mode block -- npm test
```

## 5. Use Different Scopes

Shield supports three scopes for different environments:

```bash
# Development (default)
dotset secrets set API_KEY "dev-key"

# Staging
dotset secrets set API_KEY "staging-key" --scope staging

# Production
dotset secrets set API_KEY "prod-key" --scope production

# Run with specific scope
dotset run --scope production -- npm start
```
