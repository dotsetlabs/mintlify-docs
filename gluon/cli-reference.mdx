# Gluon CLI Reference

> **Recommended**: Use the unified `dotset` CLI for the full platform experience. The standalone `gln` command is available when installing `@dotsetlabs/gluon` directly.

## Core Commands

### dotset scan

Run static security analysis.

```bash
dotset scan [options]
```

| Option | Description |
|--------|-------------|
| `--json` | Output results as JSON |

**What it checks:**
- `.env` files that should be gitignored
- Hardcoded secrets in package.json scripts
- Common security misconfigurations

### dotset run

Run a command with runtime monitoring.

```bash
dotset run [options] -- <command>
```

| Option | Description |
|--------|-------------|
| `--mode <mode>` | Secret protection: `detect`, `redact`, `block` |
| `--redact-text <text>` | Custom redaction text (default: `[REDACTED]`) |
| `--no-secrets` | Disable secret injection (Axion) |
| `--no-monitor` | Disable runtime monitoring (Gluon) |

**Examples:**

```bash
dotset run -- npm start
dotset run --mode redact -- npm start
dotset run --mode block -- node server.js
```

### dotset sbom

Generate a Software Bill of Materials.

```bash
dotset sbom [options]
```

| Option | Description |
|--------|-------------|
| `--format <fmt>` | Format: `cyclonedx`, `spdx` (default: cyclonedx) |
| `-o, --output <file>` | Output file path |
| `--static` | Generate from package.json dependencies |

**Examples:**

```bash
dotset sbom --static
dotset sbom --static --format spdx --output sbom.json
```

## Protection Modes

Gluon supports three protection modes for secret leak detection:

| Mode | Behavior |
|:-----|:---------|
| `detect` | Log exposure warnings but allow output (default) |
| `redact` | Replace detected secrets with `[REDACTED]` |
| `block` | Suppress output lines containing secrets entirely |

```bash
# Development: detect and warn
dotset run --mode detect -- npm run dev

# Staging: redact for safety
dotset run --mode redact -- npm run test

# Production: block leaks entirely
dotset run --mode block -- npm start
```

## What Gluon Monitors

### Secret Patterns
- Stripe API keys (live and test)
- AWS access keys
- GitHub tokens (PAT, OAuth, fine-grained)
- Google API keys
- JWT tokens
- Generic API keys, secrets, and passwords

### Tracked Environment Variables
Gluon also monitors for any values from environment variables you specify, detecting if those values appear in output.

### Network Connections
When enabled, Gluon tracks all outbound HTTP/HTTPS connections and can alert on connections to new/unknown domains.
