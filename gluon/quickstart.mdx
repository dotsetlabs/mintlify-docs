# Gluon Quickstart

**Runtime Security Telemetry.** Monitor secrets exposure, track network activity, and generate SBOMs.

## Installation

### CLI

```bash
npm install -g @dotsetlabs/cli
```

### SDK (for programmatic access)

```bash
npm install @dotsetlabs/gluon
```

## Initialize a Project

```bash
dotset init --gluon
```

This creates `.dotset/gluon/` with configuration files.

## Static Analysis

Analyze your codebase for security issues:

```bash
dotset scan
```

Detects:
- Exposed `.env` files that should be gitignored
- Hardcoded credentials in package.json scripts
- Security misconfigurations

## Runtime Monitoring

Run your app with monitoring:

```bash
dotset run -- npm start
```

Monitors:
- Secret exposure in stdout/stderr
- Outbound network connections
- Module loading patterns

## Secret Protection Modes

```bash
# Default: Detect and warn
dotset run -- npm start

# Redact secrets in output
dotset run --mode redact -- npm start

# Block output containing secrets
dotset run --mode block -- npm start
```

## Generate SBOM

Create a Software Bill of Materials:

```bash
dotset sbom --static
dotset sbom --static --format spdx --output sbom.json
```
