# Tachyon CLI Reference

> **Note**: CLI access requires the unified `@dotsetlabs/cli` package. The `@dotsetlabs/tachyon` package provides SDK access only.

## Core Commands

### dotset share

Expose a local port via a secure tunnel.

```bash
dotset share <port> [options]
```

| Option | Description |
|--------|-------------|
| `<port>` | Local port to expose (e.g., `3000`) |
| `--public` | Allow public access (no auth required) |
| `--subdomain <name>` | Custom subdomain name (Pro) |
| `--host <host>` | Local host to forward to (default: `localhost`) |
| `--allow <emails...>` | Allow specific email addresses |
| `--project <id>` | Link to a specific project |
| `--inspect` | Enable request inspector (localhost:4040) |

**Examples:**

```bash
dotset share 3000
dotset share 8080 --public
dotset share 3000 --subdomain my-app
dotset share 3000 --allow colleague@example.com
dotset share 3000 --inspect
```

### dotset tunnels

List active tunnels.

```bash
dotset tunnels
```

## Webhook Commands

### dotset webhooks list

List captured webhook requests.

```bash
dotset webhooks list [options]
```

| Option | Description |
|--------|-------------|
| `--limit <n>` | Number of webhooks to show |

### dotset webhooks show

Show details of a specific webhook.

```bash
dotset webhooks show <id>
```

### dotset webhooks replay

Replay a webhook request to localhost.

```bash
dotset webhooks replay <id> [options]
```

| Option | Description |
|--------|-------------|
| `-p, --port <port>` | Local port to replay to |

## Project Commands

### dotset project list

List your projects.

```bash
dotset project list
```

### dotset project show

Show project details.

```bash
dotset project show <id>
```

### dotset project link

Link current directory to a cloud project.

```bash
dotset project link <id>
```

### dotset project unlink

Unlink from cloud project.

```bash
dotset project unlink
```
