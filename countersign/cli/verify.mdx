# countersign verify

Verify the provenance and authenticity of an MCP server package before installation.

## Usage

```bash
countersign verify [package-name] [options]
```

## Arguments

| Argument | Description |
| -------- | ----------- |
| `package-name` | npm package name to verify (e.g., `@modelcontextprotocol/server-filesystem`) |

## Options

| Option | Description |
| ------ | ----------- |
| `--detailed` | Show full trust score breakdown |
| `--min-score <n>` | Minimum trust score to pass (default: 50) |
| `--require-signature` | Fail if package lacks cryptographic signature |
| `--registry <url>` | npm registry URL (default: https://registry.npmjs.org) |
| `-o, --output` | Output format: `terminal`, `json` |

## Examples

### Basic Verification

```bash
countersign verify @modelcontextprotocol/server-filesystem
```

Output:
```
✓ @modelcontextprotocol/server-filesystem@1.0.0
  Trust Score: 85/100 (HIGH)
  Publisher: anthropic (verified)
  Signature: Valid (SIGSTORE)
```

### Detailed Trust Score

```bash
countersign verify --detailed @modelcontextprotocol/server-filesystem
```

Output:
```
@modelcontextprotocol/server-filesystem@1.0.0

Trust Score Breakdown:
  ├─ Publisher Reputation    35/40
  │    └─ Verified org: anthropic
  ├─ Signature Status        25/25
  │    └─ SIGSTORE signature valid
  ├─ Provenance Data         20/20
  │    └─ Full build provenance
  └─ Package Hygiene         5/15
       └─ 2 dependencies, no install scripts

Overall: 85/100 (HIGH)
```

### JSON Output

```bash
countersign verify --output json @some/package
```

```json
{
  "package": "@some/package",
  "version": "1.0.0",
  "trustScore": 85,
  "trustLevel": "HIGH",
  "publisher": {
    "name": "anthropic",
    "verified": true
  },
  "signature": {
    "valid": true,
    "type": "SIGSTORE"
  }
}
```

### Enforce Minimum Score

```bash
# Fail if trust score is below 70
countersign verify --min-score 70 @some/package
```

## Trust Score Levels

| Score | Level | Recommendation |
| ----- | ----- | -------------- |
| 80-100 | HIGH | Safe to install |
| 50-79 | MEDIUM | Review before installing |
| 20-49 | LOW | Exercise caution |
| 0-19 | CRITICAL | Do not install |
