# Trust Scoring

Countersign calculates a trust score (0-100) for every package based on multiple signals. This score helps you make informed decisions about what to install.

## Score Components

The trust score is composed of four weighted components:

| Component | Max Points | Weight |
| --------- | ---------- | ------ |
| Publisher Reputation | 40 | 40% |
| Signature Status | 25 | 25% |
| Provenance Data | 20 | 20% |
| Package Hygiene | 15 | 15% |

## Publisher Reputation (40 points)

Who published this package?

| Signal | Points |
| ------ | ------ |
| Verified organization | +35 |
| User-trusted publisher | +30 |
| 2FA enabled | +5 |
| Long-standing account (>1 year) | +3 |
| Multiple packages | +2 |
| Unknown/new publisher | 0 |

## Package Hygiene (15 points)

Does the package follow security best practices?

| Signal | Points |
| ------ | ------ |
| No install scripts | +5 |
| Minimal dependencies (<5) | +5 |
| No network calls in postinstall | +3 |
| Clean dependency tree | +2 |
| Has install scripts | -5 |
| Excessive dependencies | -3 |

## Customizing Thresholds

Set minimum score requirements:

```bash
# Require HIGH trust (80+)
countersign verify --min-score 80 @some/package

# Set in CI/CD
COUNTERSIGN_MIN_SCORE=70 countersign verify @some/package
```

## Trusted Publishers Override

Packages from trusted publishers get a reputation bonus:

```bash
# Add trusted publisher
countersign trust add anthropic

# Their packages now start with +35 reputation
```
